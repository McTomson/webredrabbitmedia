# Leistungen — Informationsarchitektur (Entscheidung 19.07.2026)

Grundlage: Wettbewerbsanalyse neuzeitwerber.de (Voll-Crawl) + eigene Strategie
(LEISTUNGEN_ZUKUNFT_2026-07.md). Diese Datei ist die verbindliche Struktur-
Entscheidung fuer den Leistungs-Strang. Nicht neu aufrollen ohne neuen Beleg.

## Entschiedene Struktur (Thomas 19.07.): Hub + zwei Unterseiten

```
LEISTUNGEN  (Hub — Haus-Stil wie ueber-uns, schlank, in 10 Sek verstaendlich)
  rahmt beide Haelften:
   1) Deine Website        — individuell gebaut, Fixpreis, gehoert dir
   2) Talos = dein Dashboard — der mitarbeitende Teil; Basis in jeder
      Website inklusive; mehr Faehigkeiten monatlich mietbar (skaliert mit Paket)
   + Beweis (echte Artikel-Maschine, live articleCount)
   + Frag-Talos (Fragebogen)
   + CTA "kostenloser Entwurf"
     |
     |-- /leistungen/website   der Bau (neu erstellen, Relaunch ...)
     +-- /leistungen/talos     der Agent (Faehigkeiten, Freigabe, ein
                               Arbeitstag) = hier lebt das 3D-Erlebnis

  Preise -> eigene /preise-Seite (950 / 2.900 / ab 4.900, NIE 790).
  Preise erscheinen NICHT auf den Leistungsseiten.
```

Verworfen: (A) "Webdesign auf Hauptseite, Talos versteckt auf Unterseite" —
Talos ist unser staerkster Unterschied, darf nicht vergraben werden.
(C) "neuzeitwerber-Modell mit vielen Website-Typ-Unterseiten" — zu viel Bau/
Pflege jetzt, Gefahr, dass es sich wie neuzeitwerber anfuehlt. Spaeter moeglich.

Das bestehende 3D-Scroll-Deck (/relaunch-preview/talos-intro, TalosPresentation)
wird NICHT die Leistungs-Hauptseite. Es wandert als Erlebnis auf die Talos-
Unterseite. Der Hub selbst ist Haus-Stil (.rr-System), schnell und klar.

## Der entscheidende Unterschied zu neuzeitwerber (unser Hebel)

- neuzeitwerber besetzt "Ihre Website wird zum aktiven Mitarbeiter" fast woertlich
  (eigene Strategie warnt davor) — ihren Slogan NICHT kopieren.
- ABER: sie fragmentieren in 6 einzelne KI-Silos (KI-Website, KI-Automatisierung,
  KI-Chatbot, KI-Kundenservice ...). Unsere Luecke: EINE sichtbare Figur (Talos)
  statt Silo-Liste + ein ECHTES, laufendes Dashboard mit echtem BEWEIS (diese
  Seite, die Artikel-Maschine). Ihre Positionierung ist ein Versprechen, unsere
  ist ein Beweis.
- Ihre KI-Positionierung als Vorlage-Kniff (klauen): "Eine KI-gestuetzte Website
  muss nicht teurer sein" (Angst nehmen), + ehrliches "fang einfach an, Module
  spaeter" (passt zu unserer Ehrlichkeits-Linie).

## Von neuzeitwerber uebernehmen (bewaehrt)

1. Feste, wiederholte Seiten-Vorlage fuer alle Leistungs-Unterseiten (gleicher
   Bauplan wie ueber-uns -> Wiedererkennung, schnellere Produktion).
2. Immer sichtbarer "kostenlos"-CTA (Erstgespraech/Analyse), gleich auf jeder
   Seite — bei uns "kostenloser Entwurf ohne Vorkasse".
3. Ihr mehrstufiger Konfigurator = unser Frag-Talos (haben wir schon).
4. Ehrlicher "fuer wen sich das (noch) nicht lohnt"-Abschnitt.
5. "ab X"-Preis erst nach Gratis-Gespraech — Preise gebuendelt auf /preise.
6. Risiko-Umkehr-Sprache ("100% unverbindlich", "Angebot in 24h", "kein
   Verkaufsdruck", "Fixpreis, kein Abo-Zwang").

## Ihre Seiten-Vorlage (Referenz-Skelett, das wir vereinfachen)

Hero (Eyebrow + H1 + Sub + Trust-Chips + Doppel-CTA) -> Stat-Block ->
Konfigurator -> Leistungsgrid (nummeriert) -> Differenzierungs-Karten ->
nummerierter Prozess -> Referenzen-Carousel -> Vergleichstabelle (gewinnt immer)
-> Testimonials -> "So arbeiten wir" -> Kontakt/Formular -> Branchen-Grid ->
langer SEO-Ratgeber-Text -> FAQ -> Schluss-CTA-Banner.

Wir nehmen davon nur, was zu EINER Figur + Fixpreis + Ehrlichkeit passt (kein
Silo-Grid, keine Angeber-Vergleichstabelle in dieser Form).

Rohextrakte der neuzeitwerber-Seiten (falls Wortlaut noetig): Session-Scratchpad
`{home,website-erstellen-lassen,website-relaunch-agentur,ui-ux-design-agentur,
ki-automatisierung,ki-website,referenzen,seo-agentur}.md`.

## Hub-Bauplan (Grill-Ergebnisse 19.07., verbindlich)

Sektionsfolge Leistungs-Hub (Haus-Stil .rr, wie ueber-uns/tipps):
1. Hero — "Wir bauen deine Website. Und du bekommst Talos, der ab Tag eins fuer
   dich mitarbeitet." (Feinschliff Opus). Talos LEBENDIG im Hero (Idle/Gruss/
   Blick). Doppel-CTA: "Kostenloser Entwurf" + "Frag Talos".
2. Die zwei Haelften — 1) Deine Website (individuell, Fixpreis, gehoert dir)
   -> /leistungen/website ; 2) Talos = dein Dashboard (mitarbeitender Teil,
   Basis inklusive, mehr mietbar) -> /leistungen/talos.
3. Die Bruecke — Fundament (6 Punkte knapp, in jeder Website drin) + Module
   kompakt (Schreiber / Empfang / "+ auf Anfrage", je 1 Satz). "Was Talos alles
   kann" -> /leistungen/talos.
4. Der Beweis — echtes Dashboard / Artikel-Maschine, live articleCount. Unser
   Vorsprung: Beweis statt Versprechen.
5. Frag Talos — voller 5-Fragen-Fragebogen eingebettet (Logik aus TalosPage 1:1).
6. So arbeiten wir — 3 Schritte (Erstgespraech -> Entwurf ohne Vorkasse -> live)
   + ehrlicher "fang einfach an, Module spaeter"-Satz.
7. Schluss-CTA — kostenloser Entwurf + Anrufen (nur Button) + "Preise ansehen"
   -> /preise.

Feste Entscheidungen (Grill 19.07.):
- Agent-Anker = Figur Talos + Klartext-Funktion; "Dashboard" nur als konkretes
  Ding; "Mitarbeiter" NICHT als Leit-Slogan (neuzeitwerber besetzt).
- Talos am Hub: hin und wieder (Hero lebendig), PERMANENTE Buehne nur /talos.
- Frag-Talos: voll am Hub eingebettet.
- Website-Unterseite: EINE Seite jetzt (Relaunch/One-Pager als Abschnitte),
  spaeter splitten wenn SEO ruft.
- Modul-Tiefe am Hub: kompakter Ueberblick, Tiefe auf /talos.
- Bestehendes 3D-Deck (components/relaunch/talos/TalosPresentation.tsx,
  Route talos-intro) wird Grundlage der /leistungen/talos-Unterseite.
- Keine Preise auf Leistungsseiten. Kein Wort "KI" sichtbar. Telefon nur Button.
  Risiko-Umkehr-Sprache wie neuzeitwerber, aber ehrlich (kein Oversell).

Bau: Hub-Entwurf zuerst auf Preview-Route zum Reagieren (Demo vor Umbau), dann
Swap in die echte /relaunch-preview/leistungen (ersetzt die alte TalosPage dort;
deren Frag-Talos-Logik + SSR-Text wandern in Hub bzw. /talos).

## KORREKTUR 19.07. (spaet) — Research-belegter Aufbau (AUTORITATIV)

Der erste Hub-Entwurf (Route leistungen-hub) war (a) generisch: von Hand .rr-
Karten statt der echten Signatur-Bauteile (Verstoss gegen UNTERSEITEN_STIL.md §0:
NIE nachbauen, IMMER SubpageHero + rr-* importieren); (b) narrativ falsch: Talos
zu frueh (Hero + zwei Panels sofort = "zwei Produkte"-Verwirrung).

Research (Framer, Durable, Linear, Arc, Intercom, Hostinger; scratchpad-Extrakte):
KERN-REGEL = Erst das Produkt (Website) beweisen bis kein Zweifel bleibt, DANN
den Helfer einfuehren, mit Namen + Beispiel statt Technik-Wort, im GLEICHEN
Bildrahmen. Helfer ist ein VERB ("Website, die mitarbeitet"), kein zweiter
Pfeiler. Hub/Seite TEAST Talos nur, /talos erklaert voll (Trennung nach Seite).

Autoritativer Sektions-Flow der Leistungsseite (echte Haus-Bauteile):
1. Hero = SubpageHero (Wort "Leistungen" zerbricht -> Figur comp1 Zahnraeder/
   Webdesign). Reines Produkt, KEIN Talos, kein "KI"-Wort. Ink-Eyebrow +
   Crimson-H1 + Sub. BrushReveal-Hook oben (Open-Loop).
2. Der Bau-Moment: die Website setzt sich aus roten Stuecken zusammen ("das bauen
   wir fuer dich", reines Handwerk, NOCH OHNE Talos). Deine rote-Stuecke-Idee.
   OFFEN: neue Morph-Figur "Website/Browser" vs. section-by-section-Reveal echter
   rr-Bloecke. Fidelity-Demo entscheiden.
3. Was du bekommst: rr-card-soft-Grid (Design, Hosting, mobil, rechtssicher,
   Formular, Grund-SEO), reine Website-Sprache.
4. Scharnier-Zeile (Crimson-Statement): "...und sie kann mehr als andere
   Websites, weil ein Helfer im Hintergrund mitarbeitet." Talos ZUERST benannt.
5. Talos, der Helfer: Beweis statt Behauptung (Anfrage nachts -> beantwortet),
   im GLEICHEN Website-Rahmen (nie eigener App-Screenshot). Link -> /leistungen/talos.
6. Beweis/Referenzen: EIN rr-quote-Teal-Moment, echte Google-Reviews, Gold-Sterne
   (Rafael Danesh, Dmitry Pashlov, Rene Rohrer; kein erfundenes aggregateRating).
7. "Mehr als eine normale Website": kompakter Vergleich (Zeit, nicht Geld).
8. rr-faq zweispaltig: "nur ein Chatbot?" / "ersetzt Talos meine Website?" /
   "brauche ich Technik-Wissen?".
9. Schluss-CTA (rr-btn-sweep--red), produktbezogen. Preise -> /preise.

Anti-Patterns (meiden): Talos/Chat als erstes Bild; Technik-Jargon (KI/LLM/
Agent/Automatisierung) sichtbar; zwei gleich grosse CTAs; Talos ohne "...auf
deiner Website"-Rueckbindung; zwei getrennte Kaufentscheidungen.

Der generische leistungen-hub-Entwurf ist damit VERWORFEN (Lehrstueck). Die echte
Leistungsseite wird nach diesem Flow mit SubpageHero + rr-* neu gebaut.
Offen fuer Copy-Phase: last30days-Skill fuer die Sprache, die bei Betrieben zieht.
