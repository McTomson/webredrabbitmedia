## Wichtig: canonical-sicher syndizieren (kein Duplicate-Content-Schaden)

Jede Zweitveröffentlichung MUSS per `rel=canonical` auf das Original auf web.redrabbit.media zeigen,
sonst rankt die Plattform statt unserer Seite.
- **Medium:** Story import (medium.com/p/import) zieht den Artikel UND setzt automatisch das canonical
  auf die Originals-URL. NICHT per Copy-Paste neu anlegen (dann fehlt das canonical).
- **LinkedIn:** KEINE Artikel-Volltexte (LinkedIn setzt kein canonical → Duplicate). Stattdessen ein
  kurzer Post mit Hook + Link auf den Original-Artikel. Der Backlink + Referral-Traffic ist der Wert.
- **Substack:** wie gehabt (Rubrik PFLICHT, Erst-Text-dann-Video) — siehe Memory.

## Kanäle + nächste Schritte

1. **LinkedIn (Thomas' Profil):** 1 Post/Woche, der den jeweils stärksten Kosten-Artikel anteasert +
   verlinkt. Beispiel-Entwurf unten. Du postest (dein Profil/deine Stimme).
2. **Medium (canonical-Import):** die 3-4 stärksten Kosten-Artikel importieren, canonical prüfen.
3. **Reddit — NUR echter Mehrwert, kein Spam:** relevante Fragen beantworten, Link nur wenn er die
   Frage wirklich beantwortet. Kandidaten-Subs: r/de_EDV, r/Austria (nur bei passender Frage),
   r/webdev / r/Wordpress (Kostenfragen). Karma/Verlauf nötig — Reddit bestraft Drive-by-Links hart.
   KEIN autonomes Posten (Account + Ton = deine Entscheidung).
4. **Newsletter/Substack:** neue Kosten-Artikel im bestehenden 2-Mail-Flow.
5. **Source-of-Sources:** wenn ein Artikel eine eigene Zahl/Erhebung hat (z.B. regionale Preisanalyse),
   diese als zitierbare Quelle pitchen (Journalisten/Blogs verlinken Primärzahlen).
